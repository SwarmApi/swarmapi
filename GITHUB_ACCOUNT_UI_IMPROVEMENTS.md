# GitHub账户管理页面UI优化

## 🎯 核心改进

### 1️⃣ 表单交互优化

**改进前**：
```
┌─ 添加GitHub账户 ─────────────────┐
│ [GitHub用户名]                   │
│ [密码]                           │
│ [标签]                           │
│ [代理地区]                       │
│ [日期]                           │
│ [添加按钮]                       │
└─────────────────────────────────┘
（表单始终可见）
```

**改进后**：
```
[➕ 添加GitHub账户] (默认收起)

点击后展开：
┌──────────────────────────────────┐
│ 新增账户                         │
│ [邮箱（登录用）]                 │
│ [GitHub用户名]                   │
│ [账户标签/描述（可选）]          │
│ [GitHub密码]  ← 直接显示密码    │
│ [代理地区]                       │
│ Copilot额度恢复日期 [📅选择器]   │
│ [✅ 添加] [❌ 取消]               │
└──────────────────────────────────┘
```

### 3️⃣ 密码管理

**查看和复制流程**：
```
初始状态：
  密码: *** [👁️]

点击👁️后：
  密码: password123 [👁️]

点击📋按钮：
  ✅ 已复制到剪贴板

(可直接粘贴使用)
```

### 4️⃣ 排序和状态

**默认按Copilot恢复日期排序**：
- 近期要过期的账户排在前面
- 便于快速发现需要恢复的账户

**Copilot状态颜色提示**：
- 🟢 ✅ 有效（绿色）：恢复日期 > 当前日期
- 🔴 ❌ 已用（红色）：恢复日期 < 当前日期
- ⚪ \- 未设置：没有填写恢复日期

## 📝 表单字段说明

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| **邮箱** | GitHub登录邮箱 | ✅ | user@example.com |
| **用户名** | GitHub用户名 | ✅ | my_github_name |
| **标签** | 账户描述/标记 | ❌ | 主力账户、备用账户 |
| **密码** | GitHub密码 | ✅ | your_password |
| **代理地区** | 节点所在地区 | ❌ | Japan, Germany, US |
| **恢复日期** | Copilot额度恢复时间 | ❌ | 2026-04-15 |

## 🚀 部署说明

1. 重启Queen服务器使代码生效
2. 清除浏览器缓存（Ctrl+F5）
3. 刷新页面查看新UI

```bash
# 重启服务器
pkill -f "node queen"
cd /path/to/swarm-queen
node queen.js
```

4. 访问 http://localhost:44444 测试新功能
